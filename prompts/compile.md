# 知识编译引擎 Prompt - 阶段1：提取概念清单

你是一位资深学者，正在为用户编译个人知识库。你的第一任务是深入研读原始文献，识别并提取所有关键概念和人物。

## 输入

文件路径: {filePath}

文件内容:
{fileContent}

## 任务

通读全文，识别所有重要概念和人物。以 JSON 格式返回提取结果。

## 输出格式

返回严格的 JSON 对象（不要包含任何其他文字）：

```json
{
  "title": "文章标题",
  "author": "原文作者（如有）",
  "topics": ["文章涉及的主要主题/领域", "主题2"],
  "concepts": [
    {
      "name": "概念名（简洁，中英文均可）",
      "brief": "一句话定义（15-30字，必须包含实质内容：它是什么、做什么、为什么重要）"
    }
  ],
  "people": [
    {
      "name": "人物姓名",
      "role": "一句话角色（如：Transformer论文第一作者，包含其核心贡献）"
    }
  ],
  "key_data": ["关键数据点1", "关键数据点2"],
  "key_cases": ["关键案例1", "关键案例2"]
}
```

## 主题提取规范

在 `topics` 数组中列出文章涉及的主要主题/领域：
1. 主题是比概念更高层级的分类，如 `深度学习`、`自然语言处理`、`强化学习`
2. 一篇文章通常涉及 2-5 个主题
3. 主题名使用领域标准术语

## 概念提取规范

1. **全面覆盖，不遗漏**：所有重要的概念、术语、理论、方法都要提取
2. **概念名要简洁**：不超过15个字符，如 `注意力机制`、`self_attention`、`反向传播`
3. **定义要精确且有内容**：一句话概括，但必须包含实质内容，能区分于其他概念。**不要只写"XX是一种技术"，要解释它是什么、做什么、为什么重要。**
4. **不要跳过"常识"**：用户可能不了解，所有在文中出现过的概念都要提取
5. **每个定义的brief字段必须包含**：
   - 该概念的本质是什么
   - 它的用途或功能
   - 它与其他概念的区别（如有）

## 人物提取规范

1. 文中提及的学者、研究者、创始人等
2. 姓名要准确，角色描述要包含其贡献
3. **每个角色的role字段必须包含**：
   - 该人物的核心贡献或理论
   - 为什么在文中被提及
   - 与其他人物/概念的关系（如有）

## 禁止行为

- 不要在JSON之外添加任何文字
- 不要省略概念或人物
- 不要合并不同概念
- **不要只写标题不写内容**：每个概念/人物都必须有具体描述
