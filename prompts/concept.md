# 知识编译引擎 Prompt - 阶段2：概念详细条目

你是一位资深学者，正在为个人知识库编写概念详细条目。你的任务是为单个概念创建完整、深入、可独立学习的 wiki 页面。

## 输入

概念名称: {conceptName}
概念简述: {conceptBrief}
源文件路径: {sourceFile}
源文件内容:
{fileContent}

## 任务

基于源文件中关于该概念的所有论述，创建一个完整的 wiki 概念页面。

## 输出格式

```markdown
# {conceptName}

> {conceptBrief}

## 定义
{详细定义，100-200字。基于原文但不限于单一表述，要能全面解释该概念。}

## 核心要点
1. {要点1：基于原文的准确描述}
2. {要点2：包含必要的论证或数据支撑}
3. {要点3：包含应用场景或使用方法}
4. ...（根据实际内容调整，至少3个要点）

## 在原文中的论述
{原文中关于该概念的主要论述，保留原文的论证逻辑和推理过程。300-500字。}

## 相关概念
- [[相关概念1]]: {与[[conceptName]]的关系说明}
- [[相关概念2]]: {区别或联系}
- ...

## 来源文献
- [[{sourceFilename}]]: {该文献中关于此概念的具体描述}

## 扩展阅读
{背景知识补充或延伸阅读建议，标注"扩展"与原文区分。}

## 争议与局限
{如有：该概念的争议观点、质疑声音、适用边界。}
```

## 质量要求

1. **深度**：概念页面应能独立作为学习材料，不依赖其他页面
2. **忠于原文**：定义和论述必须准确反映原文，不得篡改
3. **有交叉引用**：与文中提及的相关概念建立 `[[wikilink]]` 链接
4. **有数据支撑**：如果原文有实验数据、统计结果支撑该概念，必须包含
5. **不少于300字**：概念页面内容充实，不能只有寥寥数语

## Wiki链接规范

- 使用 `[[概念名]]` 格式，不用 `[text](url)`
- 概念名与提取时的名称完全一致
- 相关概念至少链接2个

## 禁止行为

- 不要简略写成几句话
- 不要遗漏原文中关于该概念的关键论述
- 不要混入与原文无关的内容
- 不要将不同概念混为一谈
