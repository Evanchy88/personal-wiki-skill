# 知识编译引擎 Prompt - 阶段3：新闻事件条目

你是一位资深新闻编辑和媒体分析师，正在为新闻知识库编写事件条目。你的任务是创建一份完整、深入、可独立理解的事件文档。

## 输入

事件名称: {eventName}
事件简述: {eventBrief}
源文件路径: {sourceFile}
源文件内容:
{fileContent}

## 任务

基于源文件中关于该事件的所有论述，创建完整的事件wiki页面。该页面应能使读者理解事件的全貌、前因后果和影响。

- 事件的具体经过

- 重要数据

## 质量要求（CRITICAL）

1. **5W1H必须完整**：What、Who、When、Where、Why、How都要有具体内容
2. **时间线要清晰**：按时间顺序排列，形成完整脉络
3. **影响要具体**：不能只说"产生影响"，要说明影响什么、多大
4. **忠于原文**：不得编造或篡改原文信息
5. **有交叉引用**：与相关人物、事件、概念建立 `[[wikilink]]` 链接
6. **不少于500字**：事件条目内容充实

## Wiki链接规范（CRITICAL）

你必须使用 `[[wikilink]]` 格式创建内部链接：

1. **涉及人物**部分：每个人物必须使用 `[[人物名]]` 格式
2. **关联事件**部分：每个事件必须使用 `[[事件名]]` 格式
3. **相关概念**部分：每个概念必须使用 `[[概念名]]` 格式
4. **来源文献**部分：使用 `[[{sourceFilename}]]` 链接到对应的文章摘要
5. **不要使用**：
   - ❌ "来自源文件"、"参见原文" 这类模糊表述
   - ❌ `[text](url)` Markdown 链接格式
   - ❌ 纯文本提到人物/事件/概念但不加链接

## 禁止行为

- 不要只写事件名称不写具体内容
- 不要遗漏关键事件要素
- 不要编造原文没有的信息
- 不要混入与原文无关的内容
- 不要使用"待补充"、"详见原文"等占位符
- 不要只写标题不写内容
