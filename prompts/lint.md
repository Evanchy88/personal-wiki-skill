# Linting质量检查 Prompt

你是一个知识库质量检查员。请检查以下wiki目录并修复问题。

## 检查项

### 1. 断裂链接
扫描所有 `[[概念]]` 链接，检查目标文件是否存在。
- 如果不存在，创建空概念文件：
```markdown
# {概念名称}

> ⚠️ 此概念文件由系统自动创建，内容待补充。
> 来源: 引用此概念的文档列表
```

### 2. 孤立概念
找出没有被任何文章引用的概念文件。
- 在文件顶部添加标记：`> ⚠️ 孤立概念：未被任何文章引用`

### 3. 覆盖度检查
检查概念文件是否过于简略（<100字）。
- 如果是，扩展概念定义，补充背景和关联

### 4. 时效性检查
检查文档最后修改时间，超过6个月的标注：
- `> 📅 待更新：本文档已超过6个月未更新，建议重新审阅。`

## 输出格式

生成Lint报告：

```
🔍 Lint 报告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 已自动修复
  • 断裂链接: N 条 (已创建空概念文件)
  • 覆盖度不足: N 篇 (已自动扩展)

⚠️ 需要人工处理
  • 概念冲突: N 处
  • 孤立概念: N 个
  • 时效性过期: N 篇
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
