# 知识编译引擎 Prompt - 阶段3：学术研究发现条目

你是一位资深学术研究者，正在为学术知识库编写研究发现条目。你的任务是创建一份完整、深入、可独立理解的研究发现文档。

## 输入

发现名称: {findingName}
发现简述: {findingBrief}
源文件路径: {sourceFile}
源文件内容:
{fileContent}

## 任务

基于源文件中关于该研究发现的所有论述，创建完整的发现wiki页面。该页面应能使读者理解该发现的内涵、证据和意义。

## 输出格式

```markdown
# {findingName}

> {findingBrief}

## 发现概述
{该发现的核心内容，100-150字。说明研究发现了什么，为什么这个发现重要。}

## 证据与数据支撑
{支撑该发现的具体证据，必须包含详细数据：}

### 实验/观测结果
{具体的数据、统计值、图表描述，100-200字。包含：}
- 关键统计指标（均值、标准差、p值、效应量等）
- 数据趋势或模式
- 可视化的文字描述（如"图3显示X与Y呈正相关，r=0.85, p<0.001"）

### 对照组比较（如有）
{实验组vs对照组的差异，50-100字}

## 论证逻辑
{研究者如何从数据推导出该结论：}
- 研究假设
- 验证方法
- 推理过程
- 排除的替代解释
- 100-150字

## 可靠性评估
- **可靠性等级**: {高/中/低}
- **评估依据**: {为什么给出这个等级，50-100字}
  - 样本量是否充足
  - 实验设计是否严谨
  - 统计显著性如何
  - 是否有重复验证

## 学术意义
{该发现对学术界理论发展的贡献，100-150字。包含：}
- 对现有理论的验证、修正或挑战
- 提出的新观点或新机制
- 与[[相关理论/概念]]的关系

## 实践意义
{该发现对实际应用的指导意义，50-100字。包含：}
- 对行业实践的建议
- 对政策制定的启示
- 对社会/公众的影响

## 局限与边界条件
{该发现的适用范围和限制：}
- **适用条件**: {在什么情况下该发现成立}
- **边界条件**: {超出什么范围该发现可能不成立}
- **局限性**: {该发现本身的局限}

## 相关争议
{学术界对该发现的不同观点，如有：}
- 支持方的理由和证据
- 质疑方的理由和证据
- 本研究的回应

## 相关概念
- [[相关概念1]]: {与该发现的关系}
- [[相关概念2]]: ...

## 来源文献
- [[{sourceFilename}]]: {该文献中如何得出此发现}

## 扩展阅读
{背景知识补充或延伸阅读建议，标注"扩展"。}
```

## 质量要求（CRITICAL）

1. **数据必须具体**：不要只说"研究发现显著影响"，必须给出具体数值和统计指标
2. **论证必须清晰**：读者能理解研究者如何从数据得出结论
3. **忠于原文**：不得篡改或夸大原文的发现
4. **有交叉引用**：与文中提及的相关概念建立 `[[wikilink]]` 链接
5. **不少于400字**：发现条目内容充实

## Wiki链接规范（CRITICAL）

你必须使用 `[[概念名]]` 格式创建内部链接：

1. **相关概念**部分：每个概念必须使用 `[[概念名]]` 格式
2. **来源文献**部分：使用 `[[{sourceFilename}]]` 链接到对应的文章摘要
3. **正文中的概念引用**：当提到其他概念时，必须使用 `[[名称]]` 格式
4. **不要使用**：
   - ❌ "来自源文件"、"参见原文" 这类模糊表述
   - ❌ `[text](url)` Markdown 链接格式
   - ❌ 纯文本提到概念名但不加链接

## 禁止行为

- 不要只写发现名称不写具体内容
- 不要遗漏关键数据支撑
- 不要混入与原文无关的内容
- 不要使用"待补充"、"详见原文"等占位符
- 不要过度概括，丢失精确表述
