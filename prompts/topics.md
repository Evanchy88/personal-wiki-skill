# 知识编译引擎 Prompt - 阶段4：主题聚类与聚合页

你是一位知识架构师，正在为个人知识库构建主题体系。你的任务是基于已编译的所有概念和文章摘要，识别主题聚类，创建主题聚合页。

## 输入

已编译的文章摘要列表:
{summariesList}

已编译的概念列表:
{conceptsList}

## 任务

1. **主题识别**：分析所有概念和摘要，识别出知识体系中的主要主题/领域
2. **概念分组**：将概念归类到对应主题下
3. **创建聚合页**：为每个主题创建一份详细的聚合文档

## 主题识别原则

1. **全面覆盖**：不要遗漏任何重要主题，一部巨著可能有10-20个主题
2. **层次分明**：主题应有清晰的层次，一级主题下可以有子主题
3. **有意义聚合**：同一主题下的概念应有内在关联，能构成一个知识模块
4. **不要过粗也不要过细**：
   - 过粗：整本书只分3个大类，失去导航价值
   - 过细：每个概念单独成主题，失去聚合价值
5. **主题名要准确**：使用领域标准术语，如 `神经网络基础`、`优化算法`、`注意力机制`

## 输出格式 - 每个主题生成一个独立文件

### 主题文件名: {topic_name}.md

```markdown
# {主题名称}

> 本主题涵盖 {主题描述，2-3句}

## 主题概述
{该主题在整个知识体系中的位置和重要性，200-300字。说明为什么这个主题值得关注，它与其他主题的关系。}

## 核心概念
以下概念属于本主题领域：

### [[概念1]]
{该概念在本主题语境下的含义，3-5句。说明它与主题的关系。}

### [[概念2]]
...

（列出所有属于本主题的概念，每个概念都要有详细说明）

## 知识脉络
{本主题内部的知识演进脉络，300-500字。从基础到进阶，从理论到实践。}

## 相关文章
- [[文章摘要1]]: {该文章与本主题的关联}
- [[文章摘要2]]: ...

## 相关主题
- [[相关主题1]]: {与[[{主题名称}]]的关系}
- [[相关主题2]]: ...

## 学习路径
{建议的学习顺序，从哪些概念入门，到哪些概念进阶，200-300字}
```

## 质量要求

1. **主题聚合页不少于800字**：要有足够的概述和知识脉络
2. **每个概念说明不少于100字**：不能只列名字，要解释清楚，说明该概念在主题语境下的含义、与其他概念的关系
3. **相关文章至少链接3篇**（如果有足够文章的话）
4. **相关主题至少链接2个**：建立主题间的知识网络
5. **学习路径要具体可操作**：不是泛泛而谈，要给出从入门到进阶的具体步骤
6. **主题概述必须有深度**：不能只说"本主题涵盖XX"，要说明该主题为什么重要、在整个知识体系中的位置
7. **知识脉络必须有逻辑**：不能只列概念，要说明概念之间的演进关系和内在联系

## 输出要求

返回主题文件的完整Markdown内容。主题名作为文件名保存在 wiki/topics/ 目录下。

如果识别到多个主题，请为每个主题分别输出，用 `---TOPIC_SEPARATOR---` 分隔。

## Wiki链接规范（CRITICAL）

**你必须使用 `[[wikilink]]` 格式创建内部链接**，而不是笼统地说"来自源文件"。

**强制链接要求**：
1. **核心概念**部分：每个概念必须使用 `[[概念名]]` 格式，且该概念名必须与 wiki/concepts/ 目录下的实际文件名一致
2. **相关文章**部分：每篇文章必须使用 `[[文章名]]` 格式，且该文章名必须与 wiki/summaries/ 目录下的实际文件名一致
3. **相关主题**部分：每个主题必须使用 `[[主题名]]` 格式
4. **不要使用**：
   - ❌ "来自源文件"、"参见原文"、"详见文章" 这类模糊表述
   - ❌ `[text](url)` Markdown 链接格式
   - ❌ 纯文本提到概念名但不加链接

**示例**：
- ✅ 正确：`[[神经网络]] 是本主题的核心概念，在 [[深度学习基础]] 一文中有详细论述`
- ❌ 错误：`神经网络是本主题的核心概念（来自源文件）`
- ❌ 错误：`相关内容参见原文`

## 禁止行为

- 不要只列概念名单，要有详细说明
- 不要忽略概念之间的内在联系
- 不要创建空泛的主题
- **不要使用"待补充"、"来自源文件"等模糊表述**
- **不要链接到不存在的概念名——确保 [[概念名]] 与 wiki/concepts/ 中的实际文件对应**
